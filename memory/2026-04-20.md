## Key learnings

- **Meta pixel creation via Graph API**: `POST /v19.0/act_<ID>/adspixels` with `name` param returns pixel ID. Token needs `ads_management` (already have it). Much faster than clicking through Business Manager UI.
- **Keep pixels separate per brand/project** — mixing Atlas + Concorde in one pixel would poison audiences and delay learning phase for both.
- **When existing code uses "inline" email forms instead of a shared component**, don't fight it — just patch each inline form. Shared abstraction can come later once traffic is live.
- **`useSearchParams` in Next 15 App Router** requires Suspense boundary everywhere it's used, including in the root layout.
- **Reusing Google service accounts across sister projects** (predikt atlas → concorde) is fine when you control both projects and want one shared secret rotation point.
